OpenAI Codex CLI

OpenAI 官方终端 AI 编程助手 · 基于 o3 / codex-mini 系列模型 · 内置沙箱执行

📦 安装与启动▼
npm install -g @openai/codex
全局安装 Codex CLI
codex
启动交互式 REPL 对话
codex "修复这个 bug"
直接执行一个任务后退出
codex --version
查看版本号
🎯 自动化模式 核心▼
codex "任务描述"
默认 交互模式 — 每步操作前都询问确认
codex --full-auto "任务描述"
全自动化 自动批准所有读写+命令执行，无需确认
codex -a "任务描述"
等价于 --full-auto 的缩写
codex --suggest "任务描述"
只读 建议模式 — 分析问题不执行修改
🧠 模型与 Provider▼
codex --model o3
使用 o3 推理模型（最强）
codex --model o4-mini
使用 o4-mini（速度快、性价比高）
codex --model codex-mini "任务"
使用 Codex 专用小模型（编码快）
codex --provider openai
指定 provider：openai / azure / ollama
📁 工作目录与配置▼
cd /项目路径 && codex
在项目目录启动，自动理解项目结构
codex --cd /其他路径
指定工作目录
codex --quiet
安静模式，减少输出
codex --history
查看历史对话
⌨️ 交互快捷键▼
快捷键功能
Enter发送消息
Shift+Enter换行（多行输入）
Ctrl+C中断当前生成
Ctrl+D退出 Codex
y / n在交互模式下批准 / 拒绝操作
📝 实用示例▼
codex "重构 utils.py，提取公共函数"
代码重构
codex "给所有函数加 docstring"
自动补充文档
codex -a "运行测试并修复失败的用例"
全自动 修复测试
codex --suggest "分析安全漏洞"
只读安全扫描
codex "写一个 Express REST API + 测试"
从零创建项目
⚙️ 环境变量与配置文件▼
变量说明
OPENAI_API_KEYOpenAI API 密钥（必须设置）
CODEX_MODEL默认模型（覆盖 --model）
CODEX_HOME配置目录（默认 ~/.codex）
~/.codex/instructions.md全局自定义指令文件
💡 安全提示

--full-auto 模式会在沙箱中自动执行命令，但仍建议在版本控制的项目中使用。首次运行建议从默认交互模式开始。

Anthropic Claude Code

Anthropic 官方终端 AI 编程助手 · Claude Opus/Sonnet 系列 · MCP 工具扩展 · 管道模式

📦 安装与启动▼
npm install -g @anthropic-ai/claude-code
全局安装 Claude Code
claude
启动交互式 REPL
claude "修复这个 bug"
直接执行单次任务
claude --version
查看版本
🎯 权限模式 核心▼
claude
默认 交互模式 — 需手动批准文件修改和命令执行
claude --allowedTools Edit Write
白名单 只自动批准指定工具
claude --dangerously-skip-permissions
跳过全部权限 全自动执行，慎用！
🧠 模型与配置▼
claude --model claude-sonnet-4-20250514
指定 Sonnet 模型（默认，速度快）
claude --model claude-opus-4-20250514
使用 Opus 最强模型（复杂任务）
claude --max-turns 20
限制自动执行轮数（防止失控）
claude config set model claude-sonnet-4-20250514
持久化默认模型配置
claude config list
查看所有配置项
📂 上下文与记忆 重要▼
CLAUDE.md
项目级 放项目根目录，自动加载（代码规范/架构/约定）
~/.claude/CLAUDE.md
全局级 所有项目通用的偏好设置
claude --append-system-prompt "你的额外指令"
追加系统提示词
claude --context-file notes.txt
注入额外上下文文件
🔗 管道与脚本模式 实用▼
echo "问题" | claude -p
管道模式 stdin 输入 → stdout 输出
claude -p "总结这段代码" < file.py
将文件内容作为输入分析
git diff | claude -p "审查代码改动"
Code Review 审查 Git diff
claude -p --output-format json "任务"
JSON 格式输出（方便脚本解析）
claude -p --output-format text "任务"
纯文本输出（无 Markdown）
🔌 MCP 工具集成▼
claude mcp serve
将 Claude Code 作为 MCP 服务器运行
claude mcp add <name> <command>
添加 MCP 工具服务器
claude mcp list
列出已注册的 MCP 工具
⌨️ REPL 交互快捷键与斜杠命令▼
快捷键 / 命令功能
Enter发送消息
Shift+Enter换行
Esc取消输入 / 中断生成
Ctrl+C中断当前生成
Ctrl+D退出
/进入斜杠命令模式
/clear清空对话上下文
/compact压缩对话历史（节省 token 费用）
/cost查看当前会话花费
/help帮助信息
/model切换模型
/memory编辑 CLAUDE.md 记忆文件
/permissions查看/修改权限设置
/quit退出 Claude Code
📝 实用示例▼
claude "解释 src/main.py 的逻辑"
代码理解
claude "重构 auth 模块，加 JWT 支持"
功能重构
claude "写单元测试覆盖 utils/ 目录"
生成测试
git diff | claude -p "审查这些改动"
Code Review
claude -p "生成 CHANGELOG" < git.log
生成变更日志
claude "找到并修复内存泄漏"
调试排查
⚙️ 环境变量▼
变量说明
ANTHROPIC_API_KEYAnthropic API 密钥（不使用 OAuth 时需要）
CLAUDE_MODEL默认模型
CLAUDE_CODE_MAX_TURNS最大自动轮数
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC禁用遥测数据上报
💡 最佳实践

在项目根目录创建 CLAUDE.md 文件，写入项目约定、代码风格、测试要求等。Claude Code 每次启动自动加载，效果远优于临时提示。用 /compact 定期压缩对话可节省 token 开销。

🏁 场景推荐▼
场景推荐原因
快速小修改🟡 都行两者都高效
大型重构🟣 Claude Code上下文窗口更大，MCP 可扩展工具链
CI/CD 集成🟣 Claude Code管道模式 -p，支持 JSON 输出
安全敏感项目🟢 Codex内置沙箱隔离，代码不直接执行
代码审查🟣 Claude Codegit diff 管道天然契合
GPT 生态用户🟢 Codex复用 OpenAI 账户和 API Key
自动化脚本🟣 Claude CodeJSON 输出 + --max-turns 控制
💡 两者可以共存！

安装在同一台机器上互不冲突，不同项目按需选择。建议都在默认交互模式下先跑几轮熟悉，再逐步尝试自动化模式。两者都支持在项目根目录放置配置文件来定制行为。
